=>What is NestJS?
NestJS is a "progressive" Node.js framework. Think of it as Node.js with a strict set of rules. 
While basic Node.js (Express) lets you write code anywhere, NestJS forces you to follow a specific Modular Architecture. 
This makes it much easier for large teams to work on the same project without the code becoming a "spaghetti" mess. It is heavily inspired by Angular, so if you know Angular, NestJS will feel like home.

3. Core Concepts (The "Big Three")
.To build anything in NestJS, you need to understand these three building blocks:

A. Controllers (The "Gatekeeper")
=>Controllers are responsible for handling incoming requests and returning responses to the client. They handle the "Routing."
=>Role: Decide which URL (e.g., /users or /products) does what.
=>Action: They don't do heavy lifting; they just receive the data and pass it to a Service.
=>Decorator: Uses @Controller().

B. Services (The "Brain")
=>Services are where the Business Logic lives.
=>Role: This is where you calculate things, talk to the database, or integrate with 3rd-party APIs (like Stripe or Google Maps).
=>Action: The Controller says "Give me all users," and the Service actually goes and finds them.
=>Decorator: Uses @Injectable().

=>Decorators in NestJS
| Decorator   | Purpose           |
| ----------- | ----------------- |
| @Controller | Create controller |
| @Get        | GET request       |
| @Post       | POST request      |
| @Put        | Update request    |
| @Delete     | Delete request    |
| @Param      | Route parameter   |
| @Body       | Request body      |
| @Query      | Query parameter   |
| @Injectable | Service provider  |
| @Module     | Define module     |


C. Modules (The "Organizer")
=>Modules are used to group related code together.
=>Role: Every app has at least one (the AppModule). You create "Feature Modules" (e.g., UserModule, OrderModule) to keep your code organized.
=>Action: It tells Nest which Controllers and Services belong together.
=>Decorator: Uses @Module().

D. Dependency Injection (Very Important)

NestJS automatically injects services into controllers.

Example:constructor(private studentService: StudentService){}

Now controller can use:
this.studentService
This is called Dependency Injection (DI).
Advantages:
Loose coupling
Easy testing
Clean code


4. How they work together (The Flow)
=>Request: A user sends a GET request to your-api.com/cats.
=>Controller: The CatsController hears the request. It says, "I need the list of cats!" and calls the CatsService.
=>Service: The CatsService goes to the database, grabs the cats, and gives them back to the Controller.
=>Response: The Controller sends that list back to the user as a JSON object

TypeScript is mandatory: Unlike basic Express, NestJS is built for TypeScript to catch errors before you even run the code

=====CLI Commands=====
=>Install 
npm i -g @nestjs/cli

=>Create project
nest new project-name

=>Generate controller
nest g controller users

=>Generate services
nest g s users

=>Generate Module
nest g module student

=>Generate resource (CRUD)
nest g resource student

| Modules              | No       | Yes      |
| Testing              | Manual   | Built-in |
